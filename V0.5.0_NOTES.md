# PTF SMM Bot v0.5.0 — State-Aware SMM Operating Brain

## Core goal
Stop accidental campaign recreation and make the bot operate by state: campaign focus, intent, target, constraints, reference assets, visual jobs, and next best action.

## Main additions
- State-aware Router before AI Router.
- Multi-campaign awareness + current focus campaign.
- Campaign locks to prevent accidental schedule/campaign overwrite.
- Visual-only pipeline with default 2 samples.
- Visual job + visual version tracking.
- Reference asset binding + clearer media intake UX.
- Mixed-media planning foundation: video + graphic + fallback logic.
- Media availability scanner logic prepared but disabled by default until folders are finalized.
- New sheets 41–50 for intents, bindings, visual jobs, decisions, focus, locks, media availability.
- Commands: /campaign_state, /current_focus, /campaigns.

## Important behavior
If a campaign already exists and user asks for poster/image/visual/reference/revision, bot must not recreate campaign, schedule, or publication plan. It should run visual-only or clarification.
