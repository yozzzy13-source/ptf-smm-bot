# v0.6.0 — Reference Intelligence + Prompt Builder

Changed files:
- src/services/mediaIntakeService.js
- src/services/telegramKeyboardService.js
- src/services/callbackActionService.js
- src/services/stateAwareRouterService.js
- src/services/visualOnlyService.js

What changed:
1. Media intake now shows the original filename/display label + internal REF ID.
2. New reference roles:
   - Player Reference
   - Player Card
   - Style / Composition Reference
   - Brand Logo Exact
   - Venue / Sponsor Logo Exact
3. PTF logo exact mode:
   - exact logos are not passed to the image model as style/identity references;
   - prompt reserves clean top-center space for overlay;
   - this prevents model from redrawing/distorting the PTF logo.
4. Prompt Builder:
   - Player 1 = left, Player 2 = right by default;
   - style references are style/composition only;
   - people from style posters cannot become match players;
   - player references beat style references.
5. New logo behavior:
   - new logo assets are not automatically global;
   - bot asks/uses button role per event/poster.

Notes:
- True deterministic logo overlay/compositing is not added yet to avoid new image-processing dependencies.
- Current mode intentionally prevents distorted logos by reserving top-center space for exact overlay.
